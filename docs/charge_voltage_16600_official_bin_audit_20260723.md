# 官方 BIN 中 16600 写回与寄存器条件复核（2026-07-23）

## 结论

官方镜像 `samples/GXxHXxx_21.200`（256 KiB，SHA-256
`34c050d30772da07ef262fc7016e0677b9b1b4cdcd90cf43d93f0f15bf6a38c2`）显示，
按 fixed-common/banked 映射核对后，`0x0522:0x0523` 只有一类可执行直接 writer：

1. `bank1:D406/D41E`：`D1D1` 老化函数把基准减去每电芯档位后写回；`0x0A54 >= 25`
   且 4S 时为 `17600 - 4*250 = 16600`。
物理 block1 中低窗口 `0x15C5D/0x15C65` 虽有形似目标写入的字节，但逻辑地址 `0x5Cxx`
低于 `0x8000`，运行时属于 fixed-common 区并映射 block0；它不是可执行的第二个 writer。
因此“始终是 16600”可由 D1D1 周期重写或 gate 失败后的旧值保留解释。

## 原始 opcode 证据

曾误判为第二个 writer 的物理字节（inactive bank1 low window）为：

```text
15C5D / bank1:5C5D: 90 05 22 F0   MOV DPTR,#0522; MOVX @DPTR,A
15C65 / bank1:5C65: 90 05 23 F0   MOV DPTR,#0523; MOVX @DPTR,A
```

该片段不可按 bank1:5Cxx 执行。真正的老化 writer 位于 banked 区：

```text
1D406 / bank1:D406: 90 05 23 F0   ; target high
1D41E / bank1:D41E: 90 05 22 F0   ; target low
```

`D1D1` 的 gate 顺序是：`0x0490.bit1`、`0x0497.bit0`、`0x0490.bit2`，最后
`0x0490.bit0`；任一失败都在最终分档/写回之前返回。失败路径不会把目标恢复为
`0x030E:0x030F`，所以旧的 16600 可以长期留在 `0x0522:0x0523`，并被队列路径复制到
`0x0836:0x0837`。

## 当前输入是否足以产生 16600

2026-07-23 01:10 的 50 ms 实时采样为 `0x0490=0x0F`、`0x0497=0x45`、`0x0741=0x81`，
所以完整 gate 均满足。`base=17600`、4S、cycle=60、Normal、stress=`0x003C` 本身均不足以
选择 250 mV/cell；但 `0x0A54` 在每次 `0x09C7` 增量时短暂变成 `0x1B`，相邻周期又恢复
`0x02`。独立读取 `0x039D=0x1B`，确认该值来自 D264 的 R7 写入。

把 desired target 临时写成 17600 后，目标在下一次监控开始前不到约 1 秒已经恢复为 16600；
这证明当前是完整 D1D1 周期重写，不是 gate 失败后的旧值锁定。只有 AC 断开、
`0x0490.bit0=0` 时，16600 才会变成不再重算的保留值。

## 对 INT1 归因的修正

把 `0x039D` 清零后，下一次周期它又自动回到 `0x1B`，且与 `0x0A54=0x1B`、`0x09C7`
增量同步。这确认当前每个 D1D1 周期的 D264 确实以 `R7=1B` 写入，而不是历史影子残留。
但 INT1 直接归因仍有二进制矛盾：有效代码只在三个 ISR 中执行 `MOV PSW,#10`；主线没有
其它 PSW 切换指令，正常复位应为 bank0。INT1 当前可见命令 `0x1110=0x10` 对应原始表项
`bank0:0657 = 00 06`，只会把 bank2 R7 置为 `0x06`，不能产生 `0x1B`；`2D.6` 特殊分支
调用 `common:14C8 -> bank0:A274` 也只是 `RET`。因此此前“INT1 已污染主线 R7”的结论不能
仅凭 ISR ABI 成立，必须先解释现场 `PSW=0x16` 的来源（可能是软重启/非硬件复位继承）。
当前可确认的是：`0x1B` 每周期由 D264 消费并写入，INT1 仍是待验证候选，不是已闭环根因。

## 后续验证

应在同一只读采样中记录 `0x030E/0x030F`、`0x0522/0x0523`、`0x0836/0x0837`、
`0x0A54/0x039D`、`0x0490`、`0x1110`。要最终闭环，需硬件单步或在固件副本中记录 D264
入口时的 PSW/R7，并覆盖 INT1 两条分支；同时不要把 inactive bank 的低窗口数据当作可执行
控制流。
