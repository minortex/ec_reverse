# 充电日志与 EC 固件交叉调查（2026-07-22）

## 结论

本次日志中的 `3096` 可以作为一次充电会话的 **EC 充入容量候选量（mAh）**，但不能当作
FCC 或当前电池总容量。日志显示它在有充电电流时从 `151` 增长到 `3096`，而电流变为
`0 mA` 后在后续约 3.8 小时内完全不变；这与固件容量积分路径一致。

当前只读快照显示：

| 字段 | 当前值 | 含义 |
|---|---:|---|
| `EC[0x0402:0x0403]` | `5200 mAh` | DesignCapacity（LE16） |
| `EC[0x0404:0x0405]` | `4900 mAh` | 主机可见 FCC/LastFullCapacity（LE16） |
| `EC[0x0342]` | `49` | FCC factor，`49 * 100 = 4900 mAh` |
| `EC[0x0388:0x0389]` | `1202` | 上一次/当前保留的充入候选残值（BE16） |
| `EC[0x038A:0x038B]` | `0x0A6C`（现场连续采样约 `2745`） | 总吞吐/另一方向累计（BE16） |
| `EC[0x03A3]` | `0` | 容量校正状态机 idle |
| `EC[0x03A1]` | `0` | 无 FCC 重新发布 pending |

因此，日志中的 `3096` 没有在这次观察中提交为新 FCC；当前 FCC 仍为 `4900 mAh`。

## 日志事实

`/tmp/statistics.log` 共 41601 条、约 5.78 小时（记录器间隔 0.5 秒）：

- SOC 从 `2%` 到 `59%` 时，`0x0388` 从 `151` 增长到 `3096`；
- 终点电流由 `272 mA` 降至 `0 mA`，SOC 当时约 `60%`；
- 此后 `0x0388=3096` 保持不变，但公开 SOC 继续从 `60%` 走到 `89%`；
- 因而公开 SOC 不能用来反推仍在充入容量。主动充电限制、Fuel Gauge 的 SOC 平滑或发布
  策略至少有一层独立于 EC 充入积分。

公开 FCC 与容量候选也明显不是同一量：`3096 != 4900`。更准确的关系是：`0x0388`
供校正状态 5 计算候选 factor，只有满足低 SOC/低压起点、满电、ChargingCurrent=0、
连续 64 次和终点候选 `<360` 等条件，才会写 `0x0342` 并重新发布 FCC。

## 原始二进制的新确认

对 `samples/disasm/main-bank1.bin` 的原始 opcode（bank1 文件偏移）复核得到：

- `0x4880`：校正状态 1 直接清零 `0x0388:0x0389`；
- `0x4898`：状态 2 可把新计算的基线写回 `0x0388:0x0389`；
- `0x496E`：状态 5 读取 `0x0388:0x0389`，生成 factor，不是把它原样发布成 FCC；
- 文件偏移 `0x5773`（逻辑 bank1 `0xD773`）：容量积分分支更新 `0x0388`；
- 文件偏移 `0x5781`：另一分支更新 `0x038A`。

`0x0388` 与 `0x038A` 的方向分支由同一积分 worker 的状态/算术门决定。当前现场连续
采样进一步区分了两者：放电时 `0x0388` 连续 5 秒保持 `1202`，而 `0x038A` 从 `2745`
增长到 `2750`。所以 `0x0388` 不能称为总库仑吞吐量；`0x038A` 才是放电仍会增长的吞吐/另
一方向累计字段。

日志结束时为另一时间点；当前 `0x0388=1202` 比日志终点的 `3096` 小，不能解释成放电
把充入量扣减。原始码明确存在状态 1 清零和状态 2 重建基线，故跨会话观察到的下降应优先
解释为校正会话被重置/重建，而不是放电积分从 `0x0388` 中扣除。

## 当前电源与老化状态

当前快照为 `EC[0x0490]=0x0E`：bit1、bit2 为 1，bit0 为 0；包电压约 `15724 mV`，
`0x09C7:0x09CA = 14 01 00 30`，应力 word 为 `0x0030`。目标与队列均为 `17600 mV`
（`0x0522:0x0523` 与 `0x0836:0x0837` 相同）。这说明当前离电状态下没有新的老化降额写回；
目标只是保留的基准/队列值。`0x09C7/0x09C8` 也没有达到高压累计的 60/60 窗口。

## 对后续实验的要求

下一轮应在同一会话同时记录 `0x03A3`、`0x03A1`、`0x0388`、`0x038A`、`0x0342`、
`0x0404:0x0405`、`0x04AB`、`0x0514`、`0x0834:0x0837`。判定 FCC 校正成功的标准是：

1. `0x03A3` 经过状态 4/5；
2. `0x03A1.bit1` 出现 pending；
3. `0x0342` 与 `0x0404:0x0405` 随后同步变化；
4. `0x0388` 在电流归零后冻结，而 `0x038A` 的方向变化与充放电极性一致。

在没有这些状态转移证据前，不应把“充到主动限制百分比”称为一次完整容量校正。

## 插拔后目标变为 `0x40D8` 的现场复核

后续只读采样看到 `0x0522/0x0523 = D8 40`，同时 `0x0836/0x0837 = D8 40`。这两个
字段是小端排列，实际目标为 `0x40D8 = 16600 mV`；把字节按 `40 D8` 直接拼接才会得到
不可能的 55360 mV。

同一快照的输入为：

```text
base 0x030E:0x030F = 17600 mV
series 0x0491[7:6] = 4 cells
cycle count = 59
stress 0x09C9:0x09CA = 0x0030
mode 0x07A6[5:4] = Normal
auxiliary 0x0A54 = 2
0x0490 = 0x0E (bit0=0)
```

这些当前输入都不足以选择 250 mV/cell：`16600 = 17600 - 4*250`，但循环、stress、
辅助量和模式均没有达到 250 档条件。因此这不是当前快照重新计算出的老化档，而是此前某次
计算留下的旧目标。原始 `D1D1` 路径在 `0x0490` 门控或后续 gate 不成立时直接返回，不会
把 `0x0522:0x0523` 恢复成 `0x030E:0x030F`；充电队列则把已有 desired target 复制到
`0x0836:0x0837`。所以插入 AC 时目标可能被重新排队为旧的 16600，拔掉 AC 后仍保持该值，
并不表示拔电期间继续产生了 250 档老化。
