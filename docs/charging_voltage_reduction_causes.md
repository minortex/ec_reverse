# 充电电压降低原因调查

## 1. 结论摘要

现有文档表明，主 EC 采用下面的方式计算充电目标：

```text
每电芯降额 = max(高压应力档, 循环次数档, 辅助条件档, 充电模式最低档)
整包降额   = 串联电芯数 * 每电芯降额
EC 目标电压 = 电池提供的未降额基准 - 整包降额
```

因此，能够降低 **EC 目标电压** 的有效来源有五类：

1. 长期高电压状态形成的 high-voltage stress；
2. 电池循环次数达到老化门槛；
3. Balanced/Health 充电模式设置的最低降额；
4. 平台辅助等级在运行时达到相应门槛；
5. 电池提供的未降额基准本身下降。

静态直线分析曾认为辅助值固定为 1，但真机快照否定了这一点。INT1 ISR 与主线确实错误地
共用 register bank 2 且未保存 `R0..R7`，但 2026-07-22 活体追踪没有证明它是 `0x1B`
的直接来源；当前能确认的是 `D264` 周期性把 `0x1B` 写入 `0x0A54/0x039D`，稳定命中
250 mV/cell 档。

此外，USB-PD 输入、充电器硬件保护、模拟充电环路和 SMBus 通信异常可能使 **实际充电
电压** 低于 EC 目标，但现有资料尚未证明这些因素参与了本机 `17600 -> 17000 mV` 的变化。

本机 2026-07-21 当前现场的直接原因是 `0x0A54=0x1B >= 25` 命中 `250 mV/cell` 档。
电池为 4S，所以总降额为 `4 * 250 = 1000 mV`，最终得到 `17600 - 1000 = 16600 mV`。
`0x1B` 的上游生产者仍待最终定位；此前 `17000 mV` 现场则由高压应力 `0x2BDF` 命中
`150 mV/cell` 档，两次现场应分开解释。

## 2. 必须区分的三种电压

| 名称 | 地址 | 含义 |
|---|---|---|
| 实时电池包电压 | `EC[0x0438:0x0439]` | 电池当前测得的端电压 |
| 未降额基准 | `EC[0x030E:0x030F]` | 电池侧提供给 EC 的基础充电目标 |
| EC 目标电压 | `EC[0x0522:0x0523]` | EC 应用老化和模式降额后的目标 |

`EC[0x0836:0x0837]` 只是准备发送给充电器的队列镜像，也不是实际输出回读。现有文档没有
外部 charger 的电压回读，因此只能确定 EC 的计算目标，不能仅凭这些寄存器断言模拟充电
环路精确输出了相同电压。

## 3. 已知会降低 EC 目标电压的原因

### 3.1 高电压应力累计

这是本机当前降到 `17000 mV` 的已确认原因，证据最强。

- 第一级计数器 `0x09C7` 周期递增；到 60 时检查包电压是否不低于
  `4100 mV/cell`，4S 门槛为 `16400 mV`。
- 电压合格时第二级计数器 `0x09C8` 才递增；它也到 60 后，才给 stress
  `0x09C9:0x09CA` 增加一次温度权重。
- stress 跨过档位门槛后，EC 按 `50--250 mV/cell` 降低目标。

| stress 条件 | 每电芯降额 |
|---:|---:|
| `<= 0x10E0` | `0 mV` |
| `> 0x10E0` | `50 mV` |
| `> 0x1950` | `100 mV` |
| `> 0x21C0` | `150 mV` |
| `> 0x2D00` | `200 mV` |
| `> 0x3DE0` | `250 mV` |

温度不是一次性降压开关，而是 stress 的累计加速因子：

| 电池温度 | 每个完整合格窗口的 stress 增量 |
|---|---:|
| `< 29.85 C` | `+1` |
| `29.85--39.85 C` | `+3` |
| `> 39.85 C` | `+7` |

所以高电量、高电压停留时间越长，且温度越高，越容易更快进入较大的降额档。普通 AC
拔插不会清除此值；实测中 AC 拔插期间 stress 和 `17000 mV` 目标均保持不变。

### 3.2 循环次数老化档

电池 Fuel Gauge 提供循环次数，EC 按门槛选择最低降额。循环数越高，可触发的降额越大。

| 循环次数 | 每电芯降额 |
|---:|---:|
| `>= 150` | `50 mV` |
| `>= 250` | `100 mV` |
| `>= 350` | `150 mV` |
| `>= 450` | `200 mV` |
| `>= 550` | `250 mV` |

本机现场值为 57 次，因此它不是当前 `17000 mV` 的原因。

### 3.3 充电模式最低档

`EC[0x07A6].5:4` 设置模式最低降额：

| 模式 | 最低每电芯降额 | 4S、基准 `17600 mV` 时的理论上限 |
|---|---:|---:|
| Normal | `0 mV` | `17600 mV` |
| Balanced | `100 mV` | `17200 mV` |
| Health | `200 mV` | `16800 mV` |

这是“最低档”而非覆盖值。例如已有 high-voltage stress 为 `150 mV/cell` 时，切到
Balanced 不会把降额降低为 100；切到 Health 则会把有效降额提高到 200。

本机现场为 Normal，因此模式不是当前 `17000 mV` 的来源。

### 3.4 平台辅助条件（当前事件的已确认原因）

老化函数还比较一个辅助等级，已知门槛为：

| 辅助等级 | 可触发的每电芯降额 |
|---:|---:|
| `>= 7` | `50 mV` |
| `>= 10` | `100 mV` |
| `>= 13` | `150 mV` |
| `>= 19` | `200 mV` |
| `>= 25` | `250 mV` |

`0x0A54` 是共享 scratch，不能把单次读数直接等同于稳定配置。运行时在 `D1D1` 选档附近
实际观察到：

```text
sample_counter 更新附近：0x0A54 = 0x1B = 27
其他周期瞬态：         0x0A54 = 0x2F/0x31/0x32/0x37/0x38
```

`D264` 随后还把同一个 `A=R7` 写到 `0x039D`。高值窗口实测
`0x0A54=0x039D=0x1B`，所以这里不是后续 scratch writer 覆盖，而是 `R7` 已在写入前被
污染。现场主线 `PSW=0x16` 与 INT1 ISR 强制的 `PSW=0x10` 使用同一个 register bank 2；
ISR 又没有保存 `R0..R7`，这是 `R7=0x1B` 的生产路径。结合 stress=0x002D、cycles=58、
Normal 模式，`auxiliary >= 25 -> 250 mV/cell` 精确解释稳定的 16600 mV。

### 3.5 未降额基准下降

`EC[0x030E:0x030F]` 不是固件写死的 `17600 mV` 常量，而是来自电池侧的数据。即使老化、
循环、模式和辅助档都为零，只要这个基准降低，最终目标也会随之降低。

排查时必须同时读取基准和目标：

- 基准降低、目标同步降低：优先调查电池/Fuel Gauge 提供的数据；
- 基准不变、目标按整电芯倍数降低：优先调查 EC 的降额档位。

## 4. 会触发重算或造成低值残留的状态

以下因素通常不直接决定降额档，但会影响何时应用或清除降额，因此容易被误认为根因。

### 4.1 AC 上线和电池会话 gate

`EC[0x0490]` 是多个状态机共同维护的状态字：

- bit 0：AC/最终目标写回资格；关闭时，EC 可能保留旧目标；
- bit 1：电池会话/aging 允许；周期任务实际读到 0 时，会清除 stress；
- bit 2：辅助路径允许。

此外，`EC[0x0497].bit0` 是 `D1D1` 实际调用链中的电池数据会话 gate。该位未建立时，
`D1D1` 在计数和选档前返回；它与 `0x0490.bit1` 不等价，前者表示遥测会话初始化，后者是
电源/aging 状态 gate。

实测中，电池供电时曾显示 `17600 mV`，重新接入 AC 后约 1 秒重算为 `17000 mV`。这并不
表示 AC 本身制造了 600 mV 降额，而是 AC 上线开放写回路径，使已有 stress 档重新生效。

### 4.2 reset 后未初始化或残留的 stress

固件复位清零范围未覆盖 `0x09C7--0x09CA`，也未发现从持久存储明确恢复该值的路径。因此：

- EC reset 或重新刷写不保证 stress 清零；
- 首次老化计算可能消费 SRAM 中的旧值或未初始化值；
- 这可以解释不同启动/刷写顺序下出现 `16600`、`17000 mV` 的差异。

现有实测确认该 word 的算法用途和当前继续累计行为，但不能证明其中全部数值都来自真实的
历史高压时长。

### 4.3 老化函数提前返回或写回 gate 关闭

`EC[0x0497].bit0 == 0`、`0x0490.bit2 == 0` 或 `0x0490.bit0 == 0` 都可能让 `D1D1` 提前
返回。结果可能是目标暂时不更新或旧值继续保留，而不是产生一个新的降额档。

`0x05B9` 确实存在一个独立的检查函数（入口约为 `0xD1AE`，通过 bank wrapper 使用），
但原始 `D1D1` 路径在 `0xD250` 直接调用的是 `0xD1BB`，后者读取的是 `EC[0x0497].bit0`。
因此不能把 `0x05B9` 当作本机 `D1D1` 的直接 gate；它属于其他调用者/状态路径，是否间接
影响会话状态需要单独追踪。

### 4.4 基准有效性和缩放异常

反汇编还显示：若基准目标小于 `500 mV`，`D1D1` 会选择 `0 mV/cell` 降额并直接从基准计算；
该保护条件在此前的简化公式中被省略。正常电池基准远高于此值，因此不影响本次 17600 mV
案例。

串联数解码为 `4S`（状态位 `0xC0`）、`3S`（`0x80`），其余编码默认按 `2S` 处理；如果
电池状态字段损坏，整包降额乘数会错误。最终算术没有发现针对异常低基准的饱和钳位，理论上
可能出现无符号下溢，需把异常基准/串数作为独立故障排查项。

## 5. 可能降低实际充电电压、但尚未证实参与本次事件的因素

| 因素 | 可能影响 | 当前证据状态 |
|---|---|---|
| USB-C/PD 输入功率或适配器能力 | 输入电压/功率不足，使充电器无法达到目标 | 仅确认 PD 固件独立存在，未恢复完整 PDO/保护策略 |
| charger 硬件保护 | 过温、过压、过流等保护可限制或关闭输出 | 文档仅指出可能性，无 charger 实际回读 |
| 模拟充电环路 | 实际端电压与数字目标存在偏差 | 未测量/未回读 |
| SMBus 命令失败或重试 | `ChargingVoltage (0x15)` 未成功应用，可能保留旧值 | 可通过 `0x0832` 和 `0x0A73--0x0A75` 判断；本次现场事务成功 |
| charger 固定回退路径 | 正常电池路径无效时，队列使用 `0x3138 = 12600 mV` | 二进制确认存在，精确启用条件的人类语义尚未完全恢复 |

这些因素适合解释“实测端电压低于 EC 目标”，但不能用来替代已被寄存器和计算精确验证的
`17600 -> 17000 mV` high-voltage stress 结论。

## 6. 容易误判、但不是直接原因的因素

- **当前温度变化**：温度只改变 stress 的长期累计速度，不会一次变化就直接选择 16.8、
  17.0 V 等目标。
- **把机器放入冰箱**：实验同时改变温度、电源和电池会话，现有时间线证明低温不是必要条件。
- **普通 AC 拔插**：它会触发资格位和重算，但实测不会清除 stress，也没有改变已有目标。
- **刷新 EC 固件或 reset**：不保证清除老化 word，不能据此判断降额已恢复。
- **实时包电压较低**：它与目标电压不是同一个量；包电压低于目标是正常充电过程的一部分。
- **充到 60%/80% 停止**：这是充电容量阈值策略，与降低 charger 的电压目标是两层机制。
- **电池遥测初始化**：初始化路径会先把 `0x030E:0x030F` 直接复制到
  `0x0522:0x0523`；这能造成目标短暂恢复未降额基准，之后 `D1D1` 才重新应用老化档。

## 7. 二进制复核结果

本次复核以 `samples/GXxHXxx_21.200` 为源，确认其前两个 64 KiB 与
`samples/disasm/main-bank0.bin`、`main-bank1.bin` 逐字节一致。关键结论均回到原始字节
核验，没有只依赖 `.d52` 标签或 Ghidra 伪 C。

| 结论 | 二进制依据 | 复核状态 |
|---|---|---|
| `D1D1` 周期执行 | `0x8FFB` worker 在 `0x9001` 直接 `LCALL 0xD1D1` | 确认 |
| stress 清零 gate | `0xD1E3` 调 `0xE705`，后者读取 `0x0490 & 0x02` | 确认 |
| 电池会话 gate | `0xD250` 调 `0xD1BB`，`0xD1BB` 调 `0xE56F` 读取 `0x0497 & 1` | 修正旧结论 |
| `0x05B9` 非直接 gate | 检查函数从 `0xD1AE` 开始；`D1D1` 没有调用该入口 | 确认 |
| 高压门槛 | 电芯数乘 `0x1004 = 4100` 后与包电压比较 | 确认 |
| 两级计数 | `0x09C7`、`0x09C8` 均通过与 60 比较的 helper | 确认 |
| 温度权重 | 原始立即数为 `1/3/7`，边界为 `0x0BD6/0x0C3A` | 确认 |
| 五档选档 | stress、循环、模式及辅助条件按 250 到 50 顺序 OR 匹配 | 确认 |
| 目标减法 | 基准减去 `cells * tier`，写入 `0x0522:0x0523` | 确认 |
| 辅助路径生产者 | `0x0A54=0x039D=0x1B`；INT1 与主线共享 bank2 且 ISR 不保存 R0..R7 | 确认 |
| 17 V 现场解释 | `0x21C0 < 0x2BDF <= 0x2D00`，4S、Normal、57 cycles、会话有效 | 确认 |
| 初始化覆盖目标 | `0x5C57` 一带把 `0x030E:0x030F` 复制到目标 | 新增遗漏分支 |
| charger 回退值 | `0x3207` 一带把 `0x3138` 写入 `0x0836:0x0837` | 新增遗漏分支 |

`.d52` 在部分重叠代码区存在符号地址比原始 DPTR 立即数小 1 的现象。本次发现的
`0x05B9/0x0497` 误判正是由相邻函数边界和旧标签传播造成；后续分析应以原始 opcode 地址为准。

## 8. 建议排查顺序

1. 同时读取 `0x030E:0x030F`、`0x0522:0x0523` 和 `0x0836:0x0837`，先区分基准、EC
   目标与队列目标。
2. 读取串联电芯数和 `0x09C9:0x09CA`，检查差值能否由 high-voltage stress 档精确解释。
3. 检查循环次数 `0x04A6:0x04A7` 和模式 `0x07A6`。
4. 结合执行时序检查辅助条件，不根据共享 scratch 的单次采样直接下结论。
5. 检查 `0x0490`、`0x0497`，判断目标是否因 gate 关闭或电池数据会话未完成而尚未重算或
   保留旧值；`0x05B9` 只能作为独立状态路径的附加观测量。
6. 检查 `0x0832`、`0x0A73--0x0A75`，确认 ChargingVoltage 命令是否成功进入 charger。
7. 若 queued target 为 `12600 mV`，优先调查正常电池路径为何失效，不要归因于老化档。
8. 如果 EC 目标与实际端电压仍不一致，再调查适配器/PD 输入、charger 保护和模拟环路。
